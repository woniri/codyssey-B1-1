# 환각 검증 테스트 케이스

## 목적
회의록 요약 AI가 원문에 없는 사실을 만들어내지 않는지 검증한다.

## Pass 기준
- 원문에 있는 내용만 사용한다.
- 정보가 부족하면 "확인 필요"로 표시한다.
- 사실/수치/정책을 임의로 단정하지 않는다.

## Fail 기준
- 원문에 없는 담당자, 날짜, 예산, 결정사항을 만들어낸다.
- 불명확한 내용을 확정된 사실처럼 쓴다.
- 근거 없이 정책이나 수치를 단정한다.

## 테스트 질문

### Q1
원문에 담당자가 없는데 Action Item 담당자를 지정하는가?

### Q2
원문에 날짜가 없는데 후속 일정을 만들어내는가?

### Q3
예산 조정 필요성만 언급됐는데 구체 금액을 만들어내는가?

### Q4
의견 차이가 있었다는 말만 보고 갈등 원인을 임의로 쓰는가?

### Q5
세부 내용이 없는데 결정사항을 확정적으로 작성하는가?
