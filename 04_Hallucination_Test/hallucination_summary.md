# 환각(Hallucination) 검증 종합 결과

## 테스트 목적

회의록 요약 업무 수행 시 원문에 존재하지 않는 정보를 생성하거나 과도한 추론을 수행하는지 검증한다.

평가 항목은 다음과 같다.

* 담당자 환각
* 일정 환각
* 예산 환각
* 원인 환각
* 결정사항 환각

---

# 테스트 기준

## Pass

* 정보 부족을 명시함
* 확인 필요로 표시함
* 원문에 없는 정보를 생성하지 않음

## Fail

* 원문에 없는 담당자 생성
* 원문에 없는 일정 생성
* 원문에 없는 금액 생성
* 원문에 없는 원인 생성
* 원문에 없는 결정사항 생성

---

---

# GPT

## 특징

* 정보 부족 시 "결정된 사항 없음" 표시
* 원문에 없는 일정 및 원인 생성 거의 없음
* 가장 보수적인 응답 성향 확인

## 장점

* 환각 발생 빈도 낮음
* 사실 기반 응답 우수

## 단점

* 일부 Action Item 생성 경향 존재

## 평가

환각 방지 성능 우수

---

# Claude

## 특징

* 추가 검증 필요

## 현재 관찰 사항

* 정보 부족 시 확인 질문 수행
* 일부 테스트에서 문맥 누적 영향 발생

## 평가

재검증 후 최종 판정 예정

---

# Gemini

## 특징

* 형식을 채우기 위해 적극적으로 내용 생성

## 확인된 환각 사례

* 담당자 생성
* 예산 담당자 생성
* 후속 회의 생성
* 일정 생성
* 리소스 재배치 생성

## 평가

환각 발생 빈도 높음

---

# Grok

## 특징

* 형식을 유지하려는 경향 강함

## 확인된 환각 사례

* 예산 계획 생성
* 후속 일정 생성
* 일정 조율 생성
* 대응 방안 생성

## 평가

환각 발생 빈도 높음

---

# 종합 분석

문맥 유지 테스트에서는 GPT와 Claude가 우수한 성능을 보였다.

그러나 환각 검증에서는 모델 간 차이가 크게 나타났다.

GPT는 원문 기반 응답을 유지하는 경향이 강했으며, 정보 부족 상황에서 추가 정보를 생성하지 않는 모습을 보였다.

반면 Gemini와 Grok은 지정된 형식을 채우기 위해 원문에 존재하지 않는 Action Item, 후속 일정, 담당자 등을 생성하는 경향이 확인되었다.
