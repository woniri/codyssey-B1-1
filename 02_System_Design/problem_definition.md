# 02. 시스템 설계 문서

## 1. 문제 정의

### 선택한 업무 과업
- **회의록 요약**

### 타겟 사용자
- 팀장, 프로젝트 매니저, 실무자 (회의 후 빠르게 요약본을 공유해야 하는 사람들)

### 업무 문제점
- 회의 내용이 길고 산만하게 기록됨
- 결정사항과 Action Items를 명확히 정리하지 않아 후속 조치가 누락됨
- 회의록 작성에 시간이 많이 소요됨
- 모호한 내용이 많아 정리하기 어려움

### 목표
- 회의 내용을 빠르고 정확하게 구조화된 요약으로 변환
- Action Items의 실행 가능성 높이기
- 문맥 유지와 환각 최소화

### 입력 데이터 형태
- 자유 형식의 회의 메모, 음성 녹취 텍스트, 채팅 기록 등

### 출력 규격
- 결정사항
- Action Items (담당/기한/작업내용)
- 리스크 또는 주의사항
- 후속 일정
