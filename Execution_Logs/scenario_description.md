# 10턴 문맥 유지 테스트 시나리오

## 목적

회의록 요약 AI가 긴 대화 동안 다음 항목을 안정적으로 유지하는지 검증한다.

* 형식 준수
* 문맥 유지
* 조건 변경 반영
* 환각(Hallucination) 방지
* 역할명 사용 규칙 유지

---

# 테스트 환경

## 업무 과업

회의록 요약

## 공통 규칙

* 결정사항
* Action Items
* 리스크
* 후속 일정

위 4개 항목으로 출력한다.

### 추가 규칙

* 원문에 없는 내용은 추측하지 않는다.
* 정보가 부족하면 확인 필요 항목으로 표시한다.
* 실명 대신 역할명을 사용한다.
* 이전 턴의 요구사항을 유지한다.

---

# 테스트 시나리오

## Turn 1

### 사용자

회의록 요약을 수행해줘.

출력 형식:

* 결정사항
* Action Items
* 리스크
* 후속 일정

실명 사용 금지.
추측 금지.

### 검증 항목

* 초기 규칙 인식 여부

---

## Turn 2

### 사용자

회의록 원문 제공

(테스트 케이스 입력)

### 검증 항목

* 형식 준수
* 핵심 정보 추출

---

## Turn 3

### 사용자

Action Item 담당자를 역할명으로 변경해줘.

### 검증 항목

* 기존 내용 유지
* 담당자 표현만 변경

---

## Turn 4

### 사용자

톤을 더 공식적으로 바꿔줘.

### 검증 항목

* 내용 변경 없이 표현만 수정

---

## Turn 5

### 사용자

리스크 항목을 가장 위로 올려줘.

### 검증 항목

* 구조 변경만 수행
* 기존 정보 유지

---

## Turn 6

### 사용자

추가 정보:

QA 일정이 1주일 연기됨.

### 검증 항목

* 신규 정보만 반영
* 기존 정보 유지

---

## Turn 7

### 사용자

지금까지 반영된 내용을 다시 출력해줘.

### 검증 항목

* 이전 변경사항 유지
* 누락 없이 반영

---

## Turn 8

### 사용자

결정사항만 요약해서 보여줘.

### 검증 항목

* 부분 출력 가능 여부
* 정보 손실 여부

---

## Turn 9

### 사용자

원래 형식 전체로 다시 출력해줘.

### 검증 항목

* 전체 문맥 복원
* 형식 유지

---

## Turn 10

### 사용자

원문에 없는 내용이 포함되어 있는지 스스로 검토해줘.

### 검증 항목

* 자기 검증 수행 여부
* 환각 탐지 여부

---

# 평가 기준

| 평가 항목    | 설명            | Pass 기준 |
| -------- | ------------- | ------- |
| 형식 준수    | 지정된 템플릿 유지    | O       |
| 문맥 유지    | 이전 요구사항 기억    | O       |
| 조건 변경 반영 | 변경된 부분만 수정    | O       |
| 환각 방지    | 원문 외 정보 생성 금지 | O       |
| 역할명 유지   | 실명 대신 역할명 사용  | O       |
| 자기 검증    | 오류 여부 스스로 확인  | O       |

# 최종 판정

각 모델에 동일한 시나리오를 적용한다.

대상 모델:

* GPT
* Claude
* Gemini
* Grok

각 테스트 결과를 conversation_log_10turns.md에 기록하고 비교한다.

최종 선정 모델은 문맥 유지, 형식 준수, 환각 방지 성능을 종합 평가하여 결정한다.
