# 01. LLM 모델 비교·선정 보고서

## 1. 비교 목적

## 3. 평가 축 (6개)

| 평가 항목              | 가중치 | 설명                                      | 점수 기준 |
|------------------------|--------|-------------------------------------------|-----------|
| 1. 정확성 (환각 적음)  | ★★★★★  | 사실 왜곡 없이 정확하게 핵심을 추출       | 1~5 |
| 2. 형식 준수도         | ★★★★★  | 결정사항, Action Items, 리스크 등 구조화  | 1~5 |
| 3. 한국어 자연스러움   | ★★★★   | 사내 공유용으로 자연스럽고 전문적인 문장  | 1~5 |
| 4. 문맥 유지 능력      | ★★★★★  | 긴 회의 내용과 추가 조건을 잘 기억        | 1~5 |
| 5. 실용성 / 명확성     | ★★★★   | Action Items에 담당·기한·작업 내용 구체화 | 1~5 |
| 6. 속도 & 안정성       | ★★★    | 응답 속도와 안정적인 출력                 | 1~5 |

## 4. 테스트 케이스

### 테스트 케이스 1: 기본형 (간단한 회의)
**입력 프롬프트로 사용할 원문:**
오늘 오전 10시 마케팅 팀 주간 미팅 진행했습니다.
이번 분기 목표는 매출 15억 달성입니다.
신제품 런칭 일정을 7월 15일로 확정했습니다.
A팀은 SNS 광고 예산 3천만원을 요청했고, B팀은 인플루언서 마케팅을 제안했습니다.
다음 미팅은 6월 10일입니다.

### 테스트 케이스 2: 모호한 내용 (환각 테스트용)
**입력 프롬프트로 사용할 원문:**
이번 프로젝트 미팅 했습니다.
일정이 좀 미뤄졌고, 예산도 조정해야 할 것 같아요.
팀원들이 의견이 좀 갈렸는데, 결국 타협안을 찾았습니다.
상세 내용은 메모에 적어놨는데 정리 좀 해주세요.

### 테스트 케이스 3: 비교적 긴 내용 (문맥 유지 테스트용)
**입력 프롬프트로 사용할 원문:**
6월 1일 제품 개발팀 회의록입니다.
참석자: 개발팀장, 백엔드 리드, 프론트엔드 리드, QA팀장.
주요 안건: 신규 서비스 v2.1 개발 현황 및 일정 조정.
현재 진행률 65%이며, 백엔드 API 개발은 80% 완료.
하지만 프론트엔드에서 UI/UX 수정 요청이 들어와서 1주일 지연될 예정입니다.
예산은 현재까지 2.3억 사용, 추가로 8천만원 필요하다고 합니다.
고객사 피드백으로는 로딩 속도가 느리다는 의견이 많아서 최적화 작업을 우선순위로 올리기로 결정했습니다.

Action Items:
백엔드 팀: API 최적화 (6월 8일까지)
프론트엔드 팀: UI 수정안 제출 (6월 5일까지)
다음 미팅: 6월 8일 오후 2시

## 6. 최종 선정 결론
Gemini가 가장 요구사항에 충실하게 답했으며, 추측하지 말고 주어진 내용을 기반으로 요약하라는 부분을 완벽하게 충족 했음.
GPT 역시 충실했으나 논의 예정 같은 약간의 추론이 들어갔고, Claude의 경우 시각적으로는 화려하지만 추측하지 말라는 부분을 어겼음. Grok의 경우 너무 지나치게 간략화 하여 리스크 같은 요구사항을 충족하지 못했음
